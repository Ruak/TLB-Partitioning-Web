# TLB Partitioning 防护效果网页演示 PRD

## 1. 项目背景

本项目用于展示在 VexRiscv 上进行 TLB_Partitioning 防护改造后的安全效果。演示网页需要同时支持两类场景：

1. 无防护模式：通过本机三个实验脚本模拟两个用户通信和第三方监听，展示攻击者利用 cache 或 TLB 侧信道窃取身份认证密钥，并伪造通信一方向另一方发送消息。
2. 有防护模式：通过网页内置 SSH 终端连接本机可访问的 FPGA 板，运行或展示 `test.c` 相关实验结果，并与 `test_no_partition.c` 的无分区结果形成对照。

网页的核心价值不是替代底层实验脚本，而是把“攻击可行、攻击后果、防护生效、对照证据”串成一个适合答辩、展示和复现实验流程的交互界面。

## 2. 目标用户

- 论文/项目答辩观众：需要快速理解 TLB_Partitioning 解决了什么问题。
- 项目评审或导师：需要看到实机 FPGA 结果，而不是只有静态截图。
- 开发者本人：需要通过统一界面启动演示、连接板卡、查看输出、导出结果。

## 3. 产品目标

- 让用户在网页左侧边栏中一键切换“设计介绍、无防护演示、有防护演示、结果对比、实验记录”。
- 在无防护模式中清晰展示三方角色：用户 A、用户 B、监听者/攻击者。
- 在无防护模式中展示攻击链路：正常通信、密钥泄露、身份伪造、消息注入。
- 在有防护模式中内置 SSH 连接窗口，连接 FPGA 板并展示 `test.c` 的运行输出。
- 将 `test_no_partition.c` 的结果作为无防护基线提取展示，用于和 FPGA 防护结果对照。
- 支持演示过程中的关键输出留痕，便于截图、导出或复盘。

## 4. 非目标

- 不在网页中重新实现 cache/TLB 攻击算法。
- 不将 SSH 服务暴露为公网远程控制平台。
- 不在浏览器端保存 FPGA 板密码、私钥或长期凭据。
- 不让观众自由执行任意高危命令；演示命令应优先使用预设按钮和白名单。
- 不把演示做成营销 landing page；首屏应直接进入可操作的实验控制台。

## 5. 信息架构

左侧固定边栏作为主导航，推荐包含以下入口：

1. 概览
   - 展示项目一句话目标、VexRiscv + TLB_Partitioning 架构示意、当前演示状态。
2. 无防护模式
   - 本机三脚本演示控制台。
   - 用户 A、用户 B、攻击者三个输出面板。
   - 攻击事件时间线。
3. 有防护模式
   - FPGA SSH 连接面板。
   - 内嵌终端窗口。
   - `test.c` 运行控制按钮。
   - `test_no_partition.c` 基线结果展示。
4. 结果对比
   - 无防护与有防护的表格/图形对照。
   - 密钥泄露状态、伪造通信是否成功、TLB 隔离效果、输出证据。
5. 实验记录
   - 本次演示日志。
   - 关键输出摘录。
   - 导出 Markdown/JSON/纯文本的入口。

## 6. 核心用户流程

### 6.1 无防护演示流程

1. 用户进入“无防护模式”。
2. 页面显示三个角色面板：通信方 A、通信方 B、监听者。
3. 用户点击“启动无防护实验”。
4. 后端依次或并行启动三个本机脚本。
5. 页面实时显示三个脚本输出。
6. 监听者面板高亮显示窃取到的身份认证密钥。
7. 用户点击“执行伪造消息”或脚本自动推进到伪造阶段。
8. B 方接收到伪造消息，页面时间线标记“攻击成功”。
9. 结果写入实验记录，并进入结果对比页作为无防护证据。

### 6.2 有防护 FPGA 演示流程

1. 用户进入“有防护模式”。
2. 页面提示连接 FPGA 板的 SSH 信息：主机、端口、用户名、连接状态。
3. 用户点击“连接 FPGA”。
4. 后端建立 SSH 会话，前端通过内嵌终端显示 FPGA shell。
5. 用户可在终端中手动操作，也可点击预设按钮运行 `test.c` 相关命令。
6. 页面实时捕获终端输出，并标记防护模式实验结果。
7. 页面同时展示 `test_no_partition.c` 的无防护基线输出，可来自预先提取的日志、文件或一次本地脚本运行结果。
8. 用户进入“结果对比”，查看防护前后差异。

## 7. 页面设计要求

### 7.1 整体布局

- 使用左侧固定边栏 + 右侧主工作区。
- 边栏宽度保持紧凑，适合演示时快速切换。
- 主工作区根据模式切换内容，不跳转到完全不同的页面风格。
- 顶部状态栏显示：当前模式、后端服务状态、FPGA SSH 状态、本机脚本状态。
- 视觉风格应偏实验控制台：清晰、可信、适合长时间观察，不做夸张装饰。

### 7.2 概览页

必需内容：

- 项目标题：VexRiscv TLB Partitioning 安全演示平台。
- 一句话说明：通过本机攻击脚本和 FPGA 实机结果，对比 TLB 分区防护前后的侧信道攻击效果。
- 架构图区域：展示用户进程、TLB/cache、攻击者、FPGA 板、网页控制台之间关系。
- 当前状态卡片：本机后端、脚本环境、FPGA 连接、最近一次实验。

### 7.3 无防护模式页

必需模块：

- 三角色面板：
  - 用户 A：发送认证消息、密钥相关日志、通信状态。
  - 用户 B：接收消息、认证判断、最终接收内容。
  - 监听者：侧信道观测、密钥恢复、伪造消息。
- 实验控制区：
  - 启动实验。
  - 停止实验。
  - 清空输出。
  - 执行伪造阶段。
- 攻击时间线：
  - 初始化通信。
  - 侧信道观测。
  - 密钥恢复。
  - 伪造身份。
  - 注入消息。
  - 攻击成功/失败。
- 关键证据区：
  - 窃取到的 key 摘要。
  - 被伪造的身份。
  - B 方接受的伪造消息。

### 7.4 有防护模式页

必需模块：

- SSH 连接表单：
  - Host。
  - Port。
  - Username。
  - 认证方式选择：密码或私钥路径。
  - 连接/断开按钮。
- 内嵌终端：
  - 基于 WebSocket 转发 SSH 输入输出。
  - 推荐使用 `xterm.js` 或同类终端组件。
  - 支持复制输出。
  - 支持终端清屏。
- FPGA 实验控制：
  - 编译 `test.c`。
  - 运行 `test.c`。
  - 拉取/记录运行结果。
  - 重新运行。
- 无防护基线结果：
  - 展示 `test_no_partition.c` 的已提取输出。
  - 标记它是“无分区/无防护基线”，避免与当前 FPGA 防护输出混淆。
- 防护结论区：
  - 是否观测到密钥泄露。
  - 是否能够完成伪造通信。
  - 与无防护结果相比的差异。

## 8. SSH 能力设计

浏览器不能直接建立原生 SSH 连接，因此需要一个本地后端作为桥接层。

推荐架构：

- 前端：React/Vue/Svelte/普通 HTML 均可，终端组件使用 `xterm.js`。
- 后端：Node.js + `ssh2`，或 Python + `paramiko` + WebSocket。
- 通信：浏览器通过 WebSocket 连接后端，后端再连接 FPGA 板 SSH。
- 会话：每个浏览器会话对应一个 SSH session，断开页面时自动关闭。

安全要求：

- SSH 目标默认只允许本地配置文件中的 FPGA host allowlist。
- 不在前端 localStorage 中保存密码。
- 私钥路径只保存在本机后端配置中，前端只选择配置名。
- 预设按钮执行的命令必须可配置、可审计。
- 演示模式下默认隐藏完整密钥，只显示摘要；需要展示时由用户手动展开。
- 后端日志中不要持久化明文密码、私钥内容或完整认证密钥。

## 9. 后端接口草案

### 9.1 实验脚本接口

- `POST /api/lab/no-partition/start`
  - 启动无防护三脚本实验。
- `POST /api/lab/no-partition/forge`
  - 推进或触发伪造通信阶段。
- `POST /api/lab/no-partition/stop`
  - 停止无防护实验。
- `GET /api/lab/no-partition/status`
  - 获取脚本运行状态。
- `GET /api/lab/no-partition/logs`
  - 获取三个角色的最近输出。

### 9.2 FPGA/SSH 接口

- `GET /api/fpga/targets`
  - 获取允许连接的 FPGA 目标列表。
- `POST /api/fpga/ssh/connect`
  - 建立 SSH 会话。
- `POST /api/fpga/ssh/disconnect`
  - 断开 SSH 会话。
- `WS /ws/fpga/terminal`
  - SSH 终端输入输出通道。
- `POST /api/fpga/run/test-partition`
  - 运行防护模式下的 `test.c` 预设命令。
- `GET /api/fpga/results/latest`
  - 获取最近一次 FPGA 实验结果。

### 9.3 结果接口

- `GET /api/results/compare`
  - 获取无防护和有防护结果对比。
- `POST /api/results/export`
  - 导出本次演示记录。

## 10. 数据模型草案

### 10.1 ExperimentRun

字段：

- `id`：实验运行 ID。
- `mode`：`no_partition` 或 `partitioned_fpga`。
- `startedAt`：开始时间。
- `endedAt`：结束时间。
- `status`：`idle`、`running`、`success`、`failed`、`stopped`。
- `summary`：本次实验摘要。
- `artifacts`：输出日志、截图、提取结果路径。

### 10.2 AttackEvidence

字段：

- `keyRecovered`：是否恢复认证密钥。
- `keyPreview`：密钥摘要或脱敏片段。
- `forgedIdentity`：伪造的通信方。
- `forgedMessage`：伪造消息内容。
- `receiverAccepted`：接收方是否接受。
- `rawLogRefs`：原始日志引用。

### 10.3 FpgaEvidence

字段：

- `targetName`：FPGA 目标名称。
- `sshConnected`：是否连接。
- `testCommand`：执行的测试命令。
- `testCResult`：`test.c` 输出摘要。
- `noPartitionBaseline`：`test_no_partition.c` 提取结果。
- `leakObserved`：是否观察到泄露。
- `forgeSucceeded`：是否伪造成功。

## 11. 结果对比展示

推荐表格字段：

| 指标 | 无防护模式 | TLB_Partitioning 防护模式 | 说明 |
| --- | --- | --- | --- |
| 认证密钥是否可恢复 | 是/否 | 是/否 | 来自监听者脚本或 FPGA 输出 |
| 是否可伪造身份 | 是/否 | 是/否 | 看 B 方是否接受伪造消息 |
| 侧信道信号稳定性 | 高/中/低 | 高/中/低 | 根据日志或统计量 |
| 实验来源 | 本机脚本 | FPGA SSH 实机 | 标明证据来源 |
| 原始输出 | 可查看 | 可查看 | 链接到日志面板 |

对比页应给出一句结论，但不能替代原始证据。例如：

> 在无防护模式下，监听者恢复认证密钥并成功伪造消息；在启用 TLB_Partitioning 的 FPGA 实机运行中，未观察到可用于伪造通信的稳定泄露。

## 12. 关键状态与异常处理

- 后端未启动：页面显示后端连接失败，并提示启动本地服务。
- 脚本路径未配置：无防护模式显示配置缺失，不允许启动。
- 脚本运行中崩溃：保留已产生日志，标记失败原因。
- FPGA 不在线：SSH 面板显示连接失败、超时或认证失败。
- SSH 断开：终端停止输入，提示可重新连接。
- 命令执行超时：允许停止命令，并保存超时前输出。
- 无 `test_no_partition.c` 基线：结果对比页显示“基线未导入”，不生成最终结论。

## 14. 验收标准

- 页面左侧边栏可以在概览、无防护模式、有防护模式、结果对比、实验记录之间切换。
- 无防护模式可以展示三个角色的实时或准实时输出。
- 无防护模式能够在页面上明确呈现“密钥泄露”和“伪造通信成功/失败”。
- 有防护模式包含可用的 SSH 终端区域，并能连接配置中的 FPGA 目标。
- 有防护模式可以展示 `test.c` 输出，以及 `test_no_partition.c` 的基线输出。
- 结果对比页能把无防护与有防护结果并排展示，并保留原始输出入口。
- 断开 SSH、脚本失败、FPGA 不在线等异常状态都有明确提示。
- 不在浏览器持久化敏感凭据。

## 16. 开放问题

- 三个无防护脚本的实际启动命令、运行顺序和退出条件需要最终确认。
- `test.c` 在 FPGA 板上的编译/运行路径需要确认，是板上编译、交叉编译后上传，还是已有二进制。
- `test_no_partition.c` 的结果来源需要确认，是实时运行、本地日志导入，还是固定样例文件。
- FPGA SSH 认证方式需要确认，推荐优先使用本地私钥配置名，而不是在网页输入密码。
- 是否需要在演示中展示完整认证密钥，需要根据答辩安全边界决定；默认建议脱敏展示。
